## 1. 产品概述

### 1.1 产品定位

一款运行在 macOS 桌面上的像素风格电子宠物应用。宠物形象为一只边牧犬，以透明浮窗形式悬浮于桌面最上层，支持鼠标交互，并具备健康提醒功能。

### 1.2 目标用户

- Mac 电脑长时间办公用户
- 喜欢像素风、电子宠物的用户
- 需要站立/喝水提醒的健康管理用户

### 1.3 核心价值

| 维度 | 描述 |
|------|------|
| 情感陪伴 | 一只会散步、会互动的小边牧在桌面陪伴用户 |
| 健康管理 | 定时提醒用户站立和喝水，培养健康习惯 |
| 轻量无感 | 透明窗口，不遮挡工作区，可穿透鼠标事件 |
| 像素美学 | 复古像素风格，使用 200x200 精灵图呈现 |

## 3. 功能需求

### F-1：桌面悬浮窗口

#### 3.1.1 窗口规格

| 属性 | 值 |
|------|-----|
| 尺寸 | 200 × 200 像素 |
| 样式 | 无边框 (frameless)、无阴影 (hasShadow: false) |
| 背景 | 完全透明 (transparent: true) |
| 层级 | 始终置顶 (alwaysOnTop: true) |
| 任务栏 | 不在 Dock 显示 (skipTaskbar → 通过 `app.dock.hide()` 实现) |
| 安全配置 | `contextIsolation: true`、`nodeIntegration: false`、`preload: preload.js` |
| 初始位置 | 主屏幕右下角 |

#### 3.1.2 渲染页面

- 页面为一个覆盖全窗口的 `<canvas>` 元素，背景透明
- CSS 必须设置 `canvas { image-rendering: pixelated; }` 以保证像素渲染锐利

---

### F-2：精灵动画系统

#### 3.2.1 精灵图规格

| 属性 | 值 |
|------|-----|
| 画布尺寸 | 1600 × 200 像素 |
| 帧数 | 8 帧 |
| 单帧尺寸 | 200 × 200 像素 |
| 布局 | 水平排列，从左到右依次为帧 0~7 |
| 格式 | PNG，透明背景 |

#### 3.2.2 动画帧定义

| 动画名 | 帧索引 | 帧率 (fps) | 循环 | 用途 |
|--------|--------|-----------|------|------|
| `idle` | [0, 1] | 2 | 是 | 待机/呼吸 |
| `walk` | [2, 3] | 4 | 是 | 行走 |
| `pounce` | [4] | 4 | 否 | 扑击（一次性） |
| `lifted` | [5] | 1 | 是 | 被拎起悬空 |
| `pet` | [6] | 1 | 是 | 被摸头 |
| `poke` | [7] | 1 | 否 | 被戳肚皮（一次性） |

#### 3.2.3 渲染要求

- JS 层必须设置 `ctx.imageSmoothingEnabled = false`
- 每个动画帧从精灵图中按 `(frameIndex * 200, 0, 200, 200)` 裁剪绘制
- 使用 `requestAnimationFrame` 驱动渲染循环

### F-3：宠物状态机

#### 3.3.1 状态定义

| 状态 | 标识符 | 描述 |
|------|--------|------|
| 待机 | `IDLE` | 宠物停留原地，播放呼吸动画 |
| 行走 | `WALKING` | 沿随机方向移动到目标位置 |
| 扑击 | `POUNCING` | 向前方扑跳，一次性动画 |
| 被拎起 | `LIFTED` | 跟随鼠标移动，处于悬空状态 |
| 被摸头 | `PETTED` | 播放眯眼享受动画 |
| 被戳肚 | `POKED` | 播放惊讶弹跳动画 |

#### 3.3.2 状态转换规则

```
IDLE ─┬─ [定时器] ──────→ WALKING ── [到达目标] ──→ IDLE
      ├─ [鼠标按下身体] → LIFTED  ── [鼠标释放] ──→ IDLE
      ├─ [鼠标摸头部] ──→ PETTED  ── [0.5s 后] ──→ IDLE
      └─ [鼠标戳肚皮] ──→ POKED   ── [动画结束] ──→ IDLE

WALKING ── [20% 概率] ──→ POUNCING ── [动画结束] ──→ IDLE
```

#### 3.3.3 状态生命周期

每个状态必须实现以下方法：

- `enter()` — 状态进入时调用，触发动画切换
- `exit()` — 状态退出时调用，清理定时器
- `update(dt)` — 每帧调用，处理状态内逻辑

状态切换通过 EventEmitter 模式通知外部。

### F-4：宠物实体

#### 3.4.1 属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `x`, `y` | number | 宠物在桌面坐标系中的位置 |
| `width`, `height` | number | 固定为 200 × 200 |
| `direction` | `'left'` \| `'right'` | 宠物朝向，影响渲染时的水平翻转 |

#### 3.4.2 对外 API

```
pet.setState(stateName)   // 切换状态
pet.moveTo(x, y)          // 移动到目标坐标
pet.update(dt)             // 每帧更新
pet.render()               // 绘制当前帧
pet.hitTest(mx, my)        // 碰撞检测
```

#### 3.4.3 碰撞区域

在 200×200 的画布坐标系内划分三个交互区域：

| 区域 | X 范围 | Y 范围 | 返回值 |
|------|--------|--------|--------|
| 头部 | 60–140 | 0–80 | `'head'` |
| 身体 | 40–160 | 80–160 | `'body'` |
| 肚皮 | 60–140 | 120–160 | `'belly'` |
| 无命中 | — | — | `null` |

### F-5：鼠标交互

#### 3.5.1 拖动宠物 (mousedown → LIFTED → mouseup)

| 步骤 | 行为 |
|------|------|
| 触发 | 鼠标在 `'body'` 区域按下 |
| 状态 | 切换至 `LIFTED`，宠物跟随鼠标移动（带轻微摇摆效果） |
| 坐标处理 | Canvas 内偏移超过窗口半宽/半高时，通过 IPC 调用 `win.setPosition()` 移动窗口，确保宠物可被拖到桌面任意位置 |
| 释放 | 鼠标释放后，带缓动动画（easing）落回桌面，恢复 `IDLE` |

#### 3.5.2 摸头 (mousemove → PETTED → 自动恢复)

| 步骤 | 行为 |
|------|------|
| 触发 | 鼠标在 `'head'` 区域移动 |
| 状态 | 切换至 `PETTED`，播放眯眼帧，宠物微微下沉 |
| 恢复 | 鼠标离开头部区域后 0.5 秒自动恢复 `IDLE` |

#### 3.5.3 戳肚皮 (click → POKED → 自动恢复)

| 步骤 | 行为 |
|------|------|
| 触发 | 鼠标在 `'belly'` 区域单击 |
| 状态 | 切换至 `POKED`，播放惊讶帧，宠物弹跳微退 |
| 恢复 | 动画播放完毕后自动恢复 `IDLE` |

### F-6：鼠标穿透

#### 3.6.1 行为描述

默认情况下，窗口透明区域必须对鼠标事件透明（可穿透到下层应用），仅当鼠标悬停在宠物的非透明像素上时才可交互。

#### 3.6.2 实现规范

| 组件 | 职责 |
|------|------|
| **main.js** | 监听 IPC 消息 `set-ignore-mouse-events`，调用 `win.setIgnoreMouseEvents(ignore, { forward: true })` |
| **preload.js** | 通过 contextBridge 暴露 `window.electronAPI.setIgnoreMouseEvents(ignore)` |
| **renderer** | 通过 `ctx.getImageData(mx, my, 1, 1).data[3]` 检测 Alpha 通道 |

#### 3.6.3 穿透控制逻辑

```
if alpha > 0  AND 当前处于 ignore 状态 → 取消穿透 (ignore = false)
if alpha === 0 AND 当前未处于 ignore 状态 → 开启穿透 (ignore = true)
```

#### 3.6.4 性能要求

- 状态缓存：仅在穿透状态**改变时**发送 IPC 消息
- 节流：`getImageData` 检测限制为每秒 10–15 次，或在 `mousemove` 事件触发时执行

### F-7：随机散步行为

#### 3.7.1 行为参数

| 参数 | 值 |
|------|-----|
| 触发间隔 | 30–120 秒（随机） |
| 行走速度 | 30–60 px/s（随机） |
| 目标位置 | 屏幕范围内，距边缘 ≥ 100px（随机） |
| 移动方式 | 通过 IPC 调用 `win.setPosition()` 平滑移动 Electron 窗口，宠物在 Canvas 中保持居中 |

#### 3.7.2 散步中扑击

| 参数 | 值 |
|------|-----|
| 触发概率 | 散步过程中 20% |
| 效果 | 向前方跳跃 30px |

#### 3.7.3 空闲小动作

| 条件 | 行为 |
|------|------|
| IDLE 超过 10 秒 | 偶尔播放摇尾巴、歪头、眨眼等动画（可后续扩展精灵帧） |

### F-8：健康提醒

#### 3.8.1 提醒规则

| 提醒类型 | 间隔 | 触发位置 |
|----------|------|----------|
| 站立提醒 | 每 60 分钟 | 主进程 `main.js` 定时器 |
| 喝水提醒 | 每 120 分钟 | 主进程 `main.js` 定时器 |

#### 3.8.2 提醒方式

| 方式 | 实现 |
|------|------|
| 系统通知 | 通过 Electron Notification API 发送 |
| 宠物动画 | 通过 IPC 通知渲染进程，宠物走几步 + 坐下 + 头顶气泡 |

#### 3.8.3 提醒文案

| 类型 | 文字内容 |
|------|----------|
| 站立提醒 | "该站起来活动一下啦！🐕" |
| 喝水提醒 | "主人记得喝水哦～💧" |

#### 3.8.4 提醒流程

1. 主进程定时器触发
2. 主进程通过 IPC 向渲染进程发送提醒事件
3. 渲染进程让宠物播放「提醒动画」：走几步 → 坐下 → 头顶弹出气泡
4. 气泡显示 10 秒后自动消失

#### 3.8.5 持久化

- 提醒时间记录使用 `electron-store` 或本地 JSON 文件
- 应用重启后，根据上次提醒时间继续计时

#### 3.8.6 IPC 接口

preload.js 必须暴露：

```
window.electronAPI.onReminder((event, type) => { ... })
```

`type` 取值为 `'stand'` 或 `'drink'`。

### F-9：对话气泡

#### 3.9.1 视觉规格

| 属性 | 值 |
|------|-----|
| 背景 | 白色，黑色 2px 边框 |
| 宽度 | 160px |
| 高度 | 自适应文字 |
| 指向 | 底部三角形指向宠物 |
| 字体 | 像素字体（内嵌 8px 像素字体或系统字体缩小） |

#### 3.9.2 位置

气泡位于宠物头顶上方：

```
bubbleX = pet.x + pet.width / 2 - bubble.width / 2
bubbleY = pet.y - bubble.height - 10
```

#### 3.9.3 动画

显示/隐藏时附带缩放弹性动画。

#### 3.9.4 API

```
SpeechBubble.show(text, duration)   // 显示气泡，持续 duration 毫秒
SpeechBubble.hide()                 // 隐藏气泡
```

支持文本自动换行。

## 7. 术语表

| 术语 | 说明 |
|------|------|
| **精灵图 (Spritesheet)** | 将所有动画帧合并在一张大图中的资源格式 |
| **待机/呼吸 (Idle)** | 宠物静止时的循环动画，帧间身体微微起伏 |
| **状态机 (FSM)** | 有限状态机，管理宠物 6 种状态的切换规则 |
| **IPC** | Electron 主进程与渲染进程间的进程间通信 |
| **contextBridge** | Electron 安全桥接 API，在隔离上下文中暴露有限接口 |
| **鼠标穿透** | 透明窗口区域不拦截鼠标事件，直接传递给下层应用 |
| **hitTest** | 坐标碰撞检测，判断鼠标位于宠物的头部/身体/肚皮区域 |
